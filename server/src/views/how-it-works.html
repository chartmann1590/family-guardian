How Family Guardian works

Family Guardian is a self-hosted family safety platform. Your server runs in your house (or on a box you own). Phones and browsers in your family talk to it directly. No third-party servers, no analytics, no cloud account.

flowchart The flow

  1. 1. An admin sets up the server by running docker compose up. The very first visit walks them through making an admin account and a "family circle".
  2. 2. The admin invites family members by tapping "Invite family" on the dashboard. That produces a one-time link + QR code that expires in 24 hours.
  3. 3. Each family member opens the link / scans the QR, picks a password, and lands on the dashboard.
  4. 4. Everyone installs the Android app from /download. The app reports background GPS to your server and gets push-style alerts via WebSocket while the app is alive (FCM optional for deep-sleep).
  5. 5. The dashboard shows everyone on a live map. SOS, chat, geofences, and check-ins all run through the same server.

lock Privacy

  • Location data is stored in a SQLite file on your server. Nothing leaves the box.
  • The Android app talks straight to your server — there is no Family Guardian cloud account.
  • Push notifications via Firebase are optional; if you don't configure them, the in-app WebSocket still delivers everything while the app is running.
  • The web session cookie is HTTP-only. Set NODE_ENV=production to require HTTPS for cookies.

person_add Adding family members

On the dashboard, tap Invite family. You'll get three things:

  • A share link — text it / AirDrop / drop it in any chat. They tap, sign up, done.
  • A QR code — for in-person setup. The recipient points their camera at the screen.
  • An 8-character code — for typing into the Android app's "I have an invite" field.

Invites are one-time and expire in 24h. You can mint as many as you need.

android Installing on Android

Two ways:

  1. Sideload from the server: open /download in the device's browser. The APK is baked into the server image. Allow your browser to install unknown sources.
  2. Build from source: open the android/ module in Android Studio, attach your phone, hit Run.

help FAQ

Does anyone other than my family see our data?

No. The server runs on your hardware; the Android app and the web client connect directly to it. Family Guardian itself has no cloud, no telemetry, no account service.

What happens if I stop the server?

The Android app keeps queueing location updates locally until it can reach the server again. Map / chat / SOS depend on the server being up.

Can I run this outside my LAN?

Yes — put a reverse proxy (Caddy, Traefik, nginx) in front with TLS. The Android app permits cleartext only for the simple LAN case; flip to HTTPS for anything beyond your house.

How do I get notifications when the app is closed?

Set up Firebase Cloud Messaging (see README.md → "Push notifications"). It's optional — if you skip it, the in-app WebSocket still covers all events while Family Guardian is running on the device.
